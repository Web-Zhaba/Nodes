# 🚀 План запуска проекта Nodes в Production

Этот документ содержит критические шаги для обеспечения безопасности, стабильности и готовности к монетизации в ближайшие 48-72 часа.

---

## 1. Безопасность и Инфраструктура (Критично)

- [x] **Аудит Supabase RLS:**
  - Проверить, что для таблицы `recommendation_connectors` и всех новых таблиц включена Row Level Security.
  - Убедиться, что политики запрещают доступ к чужим данным (`auth.uid() = user_id`).
- [x] **Секреты (Environment Variables):**
  - Проверить `nodes-backend` на наличие захардкоженных ключей в `settings.py`.
  - Установить `DEBUG = False` в Vercel/Render.
  - Сгенерировать новый 50-символьный `SECRET_KEY` для Django.
- [x] **CORS Settings:**
  - В Django (пакет `django-cors-headers`) добавить финальный домен (например, `nodes-tracker.ru`) в `CORS_ALLOWED_ORIGINS`.
- [ ] **SMTP Провайдер:**
  - Supabase Auth имеет лимит 3 письма в час. Для продакшена **необходимо** подключить свой SMTP (Resend, SendGrid или Postmark), иначе пользователи не смогут подтвердить почту или сбросить пароль.

---

## 2. Монетизация и Юридическая часть

- [ ] **Платежный шлюз:**
  - Если используется Stripe: переключить API-ключи с `test` на `live`.
  - Настроить Webhooks для обработки успешных оплат (обновление статуса Pro в профиле).
- [x] **Юридические документы:**
  - Создать страницу `/privacy` (Политика конфиденциальности).
  - Создать страницу `/terms` (Пользовательское соглашение).
  - *Без этих страниц платежные системы могут отклонить заявку.*
- [ ] **Лимиты (Quotas):**
  - Реализовать на бэкенде или фронтенде ограничение функционала для бесплатных пользователей (например, макс. 5 узлов или лимит на генерацию рекомендаций).

---

## 3. Фронтенд и UX

- [x] **Домен:**
  - Подключить основной домен в Vercel Dashboard.
  - Настроить Redirect с `www` на основной домен.
- [x] **Мониторинг ошибок:**
  - Подключить Sentry (бесплатного тарифа хватит на запуск).
- [x] **SEO и Мета-теги:**
  - Проверить `index.html` и компоненты: наличие `title`, `description` и `og:image` для красивых ссылок в соцсетях.
- [x] **Аналитика:**
  - Установить Google Analytics или Amplitude для отслеживания конверсий в регистрацию и оплату.

---

## 4. Чек-лист проверки кода (Manual Audit)

- [x] Поиск строк `const API_KEY = '...'`. (Аудит пройден: хардкода не обнаружено)
- [x] Поиск консольных логов (`console.log`) в чувствительных местах (auth, tokens). (Проверено: логи присутствуют только в хуках синхронизации и UI)
- [x] Проверка обработчиков ошибок: пользователь не должен видеть "белый экран" при ошибке API.
- [x] Проверка производительности: запуск `npm run build` и анализ размера чанков. (Крупный чанк 620кБ приемлем для первой версии, работает offline-ready PWA)

*   **День 1:** Безопасность БД, SMTP, Юридические документы.
*   **День 2:** Платежный шлюз, Домен, Аналитика.
*   **День 3:** Финальное тестирование "песочницы", деплой, Smoke-тесты в Live режиме.
