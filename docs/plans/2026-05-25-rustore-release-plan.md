# Планирование релиза в RuStore для приложения Nodes

## 1. Совместимость с iOS
**Важно:** RuStore поддерживает только Android. Пользователи iOS не смогут скачивать приложение из этого магазина.
*   **Решение для iOS:** Использовать PWA (Progressive Web App). Пользователи могут добавить сайт на главный экран.
*   **Релиз для iOS:** Требуется отдельная публикация в Apple App Store.

## 2. Подготовительные действия
1.  **Аккаунт разработчика:**
    *   Регистрация на [console.rustore.ru](https://console.rustore.ru/).
    *   Подтверждение личности (физлицо/самозанятый/юрлицо).
2.  **Маркетинговые материалы:**
    *   Иконка: 512x512px (PNG/JPG).
    *   Скриншоты: От 2 до 10 штук, формат 16:9 или 9:16.
    *   Описание: Краткое и полное описание функционала.
    *   Политика конфиденциальности: Ссылка на внешний ресурс (например, `nodes-coral.vercel.app/privacy`).

## 3. Техническая реализация (Android)

### Сборка Production-версии
1.  **Сборка веб-части:**
    ```bash
    cd nodes-frontend
    npm run build:prod
    npx cap sync android
    ```
2.  **Подпись приложения:**
    *   Использовать существующий или создать новый `keystore`.
    *   Сборка через Android Studio: **Build > Generate Signed Bundle / APK**.
    *   Рекомендуемый формат для первого релиза: **APK** (упрощает проверку).

### Конфигурация RuStore SDK (Опционально)
Для полноценной работы в экосистеме RuStore может потребоваться интеграция следующих сервисов:
*   **Обновления:** `ru.rustore.sdk:appupdate` — для уведомления пользователей о новых версиях.
*   **Пуш-уведомления:** `ru.rustore.sdk:push` — если планируется замена Firebase Cloud Messaging.
*   **Платежи:** `ru.rustore.sdk:billing` — если планируется монетизация через российские карты.

**Настройка Gradle:**
Добавить в `android/build.gradle`:
```gradle
repositories {
    maven { url "https://artifactory-external.vkpartner.ru/artifactory/maven" }
}
```

## 4. Чек-лист перед публикацией
- [ ] `appId` в `capacitor.config.ts` совпадает с ID в консоли (`com.nodes.app`).
- [ ] Версия приложения (`versionCode` и `versionName`) в `android/app/build.gradle` увеличена.
- [ ] Приложение протестировано на физическом устройстве (не эмуляторе).
- [ ] Подготовлены учетные данные для модератора (тестовый аккаунт).

## 5. Процесс модерации
*   Срок проверки: обычно от 1 до 3 рабочих дней.
*   В случае отклонения: исправить замечания и загрузить новую версию APK.
