#### Этап 1. Подготовка и проектирование (1–2 недели)

**Задачи:**

1. Спроектировать схему БД в Supabase (дополнить существующую):
    
    - таблица `recommendations` со столбцами:
        
        - `id` (PK),
            
        - `user_id` (FK → `profiles.id`),
            
        - `content_type` (video, book, course, article, product),
            
        - `title`,
            
        - `description`,
            
        - `url` (оригинальная ссылка),
            
        - `source` (YouTube, Google Books и т. д.),
            
        - `score` (релевантность, 0–100),
            
        - `affiliate_url` (реферальная ссылка, может быть NULL),
            
        - `created_at`.
            
2. Выбрать и зарегистрироваться в партнёрских сетях:
    
    - Яндекс Партнёрская сеть;
        
    - Admitad;
        
    - (опционально) Leads.
        
3. Получить ключи доступа к бесплатным API:
    
    - YouTube Data API v3;
        
    - Google Books API;
        
    - NewsAPI;
        
    - GitHub API;
        
    - Coursera API (если доступен);
        
    - Reddit API.
        
4. Определить стек технологий:
    
    - бэкенд: FastAPI (Python);
        
    - ORM: SQLAlchemy (подключение к Supabase);
        
    - фронтенд: React + Tailwind CSS;
        
    - БД: Supabase (PostgreSQL);
        
    - аутентификация: Supabase Auth;
        
    - инфраструктура: Docker, Nginx, Gunicorn.
        

#### Этап 2. Разработка бэкенда (3–4 недели)

**Задачи:**

1. Реализовать аутентификацию через Supabase Auth.
    
2. Написать модуль импорта данных:
    
    - запрос к таблицам `nodes` и `profiles` в Supabase;
        
    - извлечение `label`, `connectors`, `pulseCount`, `lastPulse`.
        
3. Реализовать алгоритм расчёта интересов:
    
    - суммирование `pulseCount` по тегам (`connectors`);
        
    - формирование профиля интересов с весами;
        
    - учёт давности активности (`lastPulse`).
        
4. Интегрировать бесплатные API:
    
    - реализовать запросы к YouTube, Google Books, NewsAPI и др. по ключевым словам из тегов;
        
    - обработать ответы, извлечь релевантные поля (заголовок, описание, ссылка, рейтинг).
        
5. Подключить партнёрские сети:
    
    - написать функцию `get_affiliate_link()` для генерации реферальных ссылок;
        
    - интегрировать API Яндекс Партнёрки и Admitad.
        
6. Реализовать модуль ранжирования:
    
    - расчёт `score = weight_tag × rating × recency_factor`;
        
    - фильтрация по критериям (рейтинг > 4,0; дата публикации < 6 месяцев).
        
7. Написать модуль сохранения рекомендаций:
    
    - вставка данных в таблицу `recommendations`;
        
    - обновление существующих рекомендаций (каждые 24 часа).

### Технические детали реализации

**Алгоритм работы системы:**

1. Пользователь авторизуется через Supabase Auth.
    
2. Бэкенд запрашивает `nodes` для `user_id`.
    
3. Рассчитывается профиль интересов.
    
4. По тегам с весом > 20 делаются запросы к API и партнёрским сетям.
    
5. Контент фильтруется и ранжируется.
    
6. Рекомендации сохраняются в `recommendations`.
    
7. Фронтенд отображает карточки, сгруппированные по тегам.
    

**Пример данных:**

- **Интересы:** `{"программирование": 75, "обучение": 45, "книги": 30}`.
    
- **Рекомендации:**
    
    - раздел «Программирование» (75 баллов): 5 видео (YouTube), 3 репозитория (GitHub), 2 товара (Яндекс Маркет);
        
    - раздел «Обучение» (45 баллов): 3 курса (Coursera), 2 статьи (NewsAPI);
        
    - раздел «Книги» (30 баллов): 2 книги (Google Books).

### Ожидаемые результаты через 6 месяцев

- **Пользователи:** 10 000 активных пользователей.
    
- **Конверсия в премиум:** ≥ 5 %.
    
- **ARPU:** ≥ $3/мес (доход с реферальных переходов + подписки).
    
- **Retention Rate (7 дней):** ≥ 40 %.
    
- **CTR рекомендаций:** ≥ 15 %.
    
- **NPS:** ≥ 60 %.

### Риски и решения

| Риск                                                         | Решение                                                                                              |
| ------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| Ограничения бесплатных API (квоты)                           | Кэширование результатов (Redis); оптимизация запросов; переход на платные тарифы при росте аудитории |
| Низкая релевантность рекомендаций                            | A/B‑тестирование алгоритмов; фидбек пользователя; дообучение модели                                  |
| Проблемы с партнёрскими сетями (отказ в реферальных ссылках) | Подключение альтернативных сетей (Leads, другие агрегаторы)                                          |
| Рост нагрузки на сервер                                      | Горизонтальное масштабирование; использование CDN для статических файлов                             |
