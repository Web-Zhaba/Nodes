### 1. Видение проекта

**Название:** Content Recommender — генератор персонализированных рекомендаций контента на основе привычек пользователя.

**Миссия:** Помочь пользователям находить релевантный контент (книги, курсы, видео, статьи) на основе их ежедневных привычек и интересов, зафиксированных в трекер‑приложении Nodes.

**Ключевая идея:** Превратить данные о привычках в персонализированные рекомендации. Если пользователь регулярно выполняет узел «Изучение Python», система предложит курсы по Python, книги по алгоритмам и видео по фреймворкам.

**Целевая аудитория:**

- пользователи трекера привычек Nodes (25–40 лет);
    
- люди, заинтересованные в саморазвитии и обучении;
    
- профессионалы, желающие систематизировать процесс обучения.
    

**Ценностное предложение:**

- **Персонализация:** рекомендации строятся на реальных действиях пользователя, а не на заявленных интересах.
    
- **Контекстность:** учитывает стабильность привычек и активность пользователя.
    
- **Экономия времени:** избавляет от поиска контента по темам.
    
- **Развитие привычек:** рекомендации подкрепляют существующие узлы (привычки).
    

---

### 2. Цели и KPI

**Бизнес‑цели:**

- привлечь 10 000 пользователей за 6 месяцев;
    
- достичь конверсии в премиум 5 %;
    
- обеспечить средний доход с пользователя (ARPU) $3/мес.
    

**Продуктовые KPI:**

- Retention Rate (7 дней): ≥ 40 %;
    
- CTR рекомендаций: ≥ 15 %;
    
- NPS (индекс лояльности): ≥ 60 %;
    
- среднее количество рекомендаций на пользователя в день: ≥ 5.
    

---

### 3. Функциональные требования

**Обязательные функции (MVP):**

1. **Импорт данных из Nodes:**
    
    - авторизация через OAuth Nodes;
        
    - автоматическая синхронизация узлов (каждые 24 часа или по Webhook);
        
    - поддержка полей: `label`, `connectors` (теги), `pulseCount`, `lastPulse`.
        
2. **Расчёт интересов пользователя:**
    
    - анализ тегов узлов и частоты их выполнения (`pulseCount`);
        
    - формирование профиля интересов с весами (например, «программирование»: 135, «карьера»: 90).
        
3. **Получение рекомендаций:**
    
    - запросы к внешним API (YouTube, Goodreads, Coursera, Medium, NewsAPI);
        
    - фильтрация дубликатов и низкорейтингового контента;
        
    - ранжирование по релевантности: `score = weight_tag × rating_content`.
        
4. **Отображение рекомендаций:**
    
    - группировка по тегам/интересам;
        
    - карточки контента с заголовком, описанием, ссылкой, источником, оценкой;
        
    - пагинация (20 рекомендаций на страницу).
        
**Простейшая аналитика:**
    
    - статистика по интересам (график «ваши топ‑5 тем»);
        
    - история рекомендаций.
        

**Дополнительные функции (после MVP):**

- фидбек от пользователя («Нравится»/«Неинтересно») для улучшения рекомендаций;
    
- уведомления о новых рекомендациях;
    
- экспорт рекомендаций в PDF/CSV;
    
- ручной ввод интересов (если нет аккаунта в Nodes);
    
- социальный функционал: делиться подборками.

### 4. Нефункциональные требования

- **Производительность:** время отклика API ≤ 2 с, загрузка дашборда ≤ 3 с.
    
- **Надёжность:** доступность 99,9 % (SLA), резервное копирование БД ежедневно.
    
- **Безопасность:** OAuth 2.0 для авторизации, HTTPS, хэширование паролей (если есть регистрация), защита от SQL‑инъекций.
    
- **Масштабируемость:** архитектура должна выдерживать рост до 100 000 пользователей.
    
- **Удобство использования:** интуитивно понятный интерфейс, подсказки для новых пользователей, адаптивный дизайн.
    
- **Поддержка браузеров:** Chrome, Firefox, Safari (последние 2 версии).

### 5. Архитектура и стек технологий

**Бэкенд:**

- язык: Python 3.11+;
    
- фреймворк: FastAPI (асинхронный, высокая производительность);
    
- ORM: SQLAlchemy;
    
- библиотеки: Pandas (анализ данных), Scikit‑learn (алгоритмы рекомендаций), Requests (HTTP‑запросы).
    

**База данных:** PostgreSQL 15+

- таблицы: `users`, `nodes`, `recommendations`, `user_preferences`, `content_sources`.
    

**Фронтенд:**

- вариант 1 (MVP): Streamlit (быстрый дашборд);
    
- вариант 2 (основной): React 18 + TypeScript + Tailwind CSS + Shadcn UI.
    

**Визуализация:** Plotly (графики интересов), D3.js (граф интересов — в перспективе).

**Инфраструктура:** Docker, Nginx, Gunicorn, Redis (кэширование).

**Внешние API:** YouTube Data API, Goodreads API, Coursera API, Medium API, NewsAPI.

### 6. Интеграция с Nodes

**Сценарий 1. Отдельный сервис**

1. Пользователь регистрируется в Content Recommender.    
2. В настройках вводит API‑ключ от Nodes или авторизуется через OAuth.
3. Сервис синхронизирует данные узлов.
4. Показывает рекомендации в своём интерфейсе.

**Сценарий 2. Модуль внутри Nodes**

1. В интерфейсе Nodes появляется вкладка «Рекомендации».
2. Бэкенд Nodes вызывает Content Recommender API.
3. Рекомендации отображаются в карточке рядом с узлами.

**Точки интеграции:**

- OAuth 2.0 (авторизация);
    
- REST API Nodes (выгрузка узлов);
    
- Webhooks (реал‑тайм‑обновления при выполнении узла).

### 7. Модель монетизации

**Стратегия:** Freemium + партнёрские программы.

**Тарифы:**

- **Free:** 5 рекомендаций в день, базовые теги, YouTube/Medium;
    
- **Premium ($5/мес):** неограниченные рекомендации, все источники (Coursera, edX, Goodreads), расширенная аналитика, экспорт в PDF;
    
- **Pro ($15/мес):** White‑label API для компаний, кастомные интеграции.
    

**Партнёрские программы:**

- комиссия за переходы на Coursera/Udemy (5–15 % от стоимости курса);
    
- партнёрство с издательствами (скидка на книги из рекомендаций).
    

**Дополнительные источники:**

- спонсорский контент («Рекомендовано партнёром»);
    
- премиум‑аналитика (отчёты «Как привычки влияют на обучение»).

### 8. Этапы разработки (Roadmap)

**Этап 1. MVP (1–2 месяца)**

- бэкенд: FastAPI + SQLite;
    
- импорт данных: ручной JSON из Nodes;
    
- рекомендации: только по тегу «программирование» из YouTube API;
    
- фронтенд: Streamlit‑дашборд;
    
- монетизация: партнёрские ссылки.
    

**Этап 2. Основной релиз (3–4 месяца)**

- полноценная БД: PostgreSQL;
    
- интеграция с Nodes через OAuth;
    
- поддержка 3+ источников контента;
    
- React‑интерфейс;
    
- запуск Freemium‑тарифов.
    

**Этап 3. Расширение (5–6 месяцев)**

- алгоритмы машинного обучения (Scikit‑learn);
    
- фидбек и A/B‑тестирование;
    
- уведомления и экспорт;
    
- партнёрские программы с издательствами.

### 9. Риски и решения

|Риск|Вероятность|Влияние|Решение|
|---|---|---|---|
|API‑лимиты внешних сервисов|Высокая|Среднее|Кэширование (Redis), пакетные запросы, приоритезация источников|
|Низкая релевантность рекомендаций|Средняя|Высокое|A/B‑тестирование алгоритмов, фидбек пользователя, дообучение модели|
|Сложности интеграции с Nodes|Средняя|Среднее|Чёткое API‑соглашение, Webhooks, резервный импорт JSON|
|Конкуренция с существующими рекомендательными сервисами|Высокая|Высокое|Фокус на уникальности: связь с привычками, а не интересами|
|Недостаток данных от новых пользователей Nodes|Высокая|Среднее|Ручной ввод интересов, рекомендации по популярным темам|

### 10. Критерии успеха

**На этапе MVP:**

- запущено API с интеграцией YouTube;
    
- синхронизация данных из Nodes работает;
    
- первые 100 пользователей получили рекомендации;
    
- CTR ≥ 10 %.
    

**Через 6 месяцев:**

- 10 000 активных пользователей;
    
- конверсия в премиум ≥ 5 %;
    
- ARPU ≥ $3;
    
- NPS ≥ 60 %.
