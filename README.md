## 🎯 Философия

**Nodes** отказывается от классического понятия "привычки" в пользу **узлов (Nodes)** — единиц действия, образующих динамическую нейронную сеть вашей жизни.

| Концепт | Описание | Пример |
|---------|----------|--------|
| **Node (Узел)** | Единица действия | "Бег", "Чтение", "Медитация" |
| **Core (Ядро)** | Гравитационный центр сферы жизни | "Здоровье", "Развитие" |
| **Connector (Связь)** | Семантический тег, связывающий узлы | "#спорт", "#утро" |
| **Pulse (Импульс)** | Зарегистрированное действие (отметка) | Выполнение привычки за сегодня |
| **Stability (Стабильность)** | Жизнеспособность связи (0–100%) | Скользящее среднее, а не жесткие стрики |

### Почему не стрики?
Обычные стрики обнуляются при первом пропуске, что демотивирует. **Nodes** рассчитывает стабильность по принципу **экспоненциального затухания**:
- Пропуск одного дня плавно снижает стабильность, а не сбрасывает её в ноль.
- Выполнение с перевыполнением (Overdrive ×1.5) ускоряет восстановление "остывших" связей.
- Узлы можно замораживать (**Crystal Nodes**), фиксируя достигнутый результат.

## 🏗 Архитектура приложения

Приложение построено по гибридной архитектуре с упором на Local-first и PWA:

```
┌────────────────────────────────────────────────────────┐
│                   Frontend (React 19)                  │
│  - Zustand 5 (Local-first с IndexedDB)                 │
│  - react-force-graph-2d (Интерактивная сеть)           │
│  - Obsidian-style Workspace (Вики-заметки)            │
└───────────────────────────┬────────────────────────────┘
                            │
            ┌───────────────┴───────────────┐
            ▼                               ▼
┌──────────────────────┐        ┌──────────────────────┐
│  Supabase Auth & DB  │        │   Django API (DRF)   │
│  - PostgreSQL 17     │◀───────│  - Расчет стабильности│
│  - Хранение данных   │        │  - Тепловая карта    │
│  - RLS Policies      │        │  - История орбиты    │
└──────────────────────┘        └──────────────────────┘
```

### Типы узлов
1. **Binary** (Да/Нет) — Фиксированный заряд. Период полураспада: 7 дней.
2. **Quantity** (Количество) — Заряд пропорционален выполнению цели. Период полураспада: 5 дней.
3. **Duration** (Время) — Заряд накапливается через встроенный таймер. Период полураспада: 10 дней.

## 🛠 Технологический стек

### Frontend (`nodes-frontend/`)
- **React 19** + **Vite 7** + **TypeScript 5.9** (strict)
- **Zustand 5** — Атомное управление локальным состоянием с синхронизацией в **IndexedDB** (`idb-keyval`)
- **Tailwind CSS v4** + **Shadcn UI** (стиль `new-york`, цвет `neutral`)
- **react-force-graph-2d** — D3-движок визуализации интерактивной нейросети
- **Capacitor** — Сборка нативного мобильного приложения для Android & iOS
- **PWA (Vite PWA)** — Поддержка офлайн-режима с баннером состояния сети
- **i18next** — Локализация (RU / EN)

### Backend (`nodes-backend/`)
- **Django 6** + **Django REST Framework** (Python 3.12)
- **Supabase PostgreSQL** — Прямое подключение в обход RLS (Service Role)
- JWT-валидация токенов Supabase Auth через JWKS

## 🚀 Запуск проекта локально

Перед запуском убедитесь, что у вас установлены **Node.js (v18+)** и **Python (3.12+)**.

### 1. Настройка базы данных (Supabase)
1. Создайте проект на [supabase.com](https://supabase.com/).
2. Примените SQL-схему из файла `docs/supabase-schema.sql` в SQL Editor на Supabase.

### 2. Запуск бэкенда (`nodes-backend/`)
1. Создайте виртуальное окружение и активируйте его:
   ```bash
   cd nodes-backend
   python -m venv venv
   # На Windows:
   venv\Scripts\activate
   # На macOS/Linux:
   source venv/bin/activate
   ```
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Создайте файл `.env` на основе примера и укажите ваши ключи подключения к БД и JWT:
   ```env
   DATABASE_URL=your_supabase_postgres_connection_string
   SUPABASE_JWT_SECRET=your_jwt_secret
   SUPABASE_URL=your_supabase_project_url
   SECRET_KEY=your_django_secret_key
   DEBUG=True
   ALLOWED_HOSTS=127.0.0.1,localhost
   ```
4. Запустите Django-сервер:
   ```bash
   python manage.py runserver
   ```

### 3. Запуск фронтенда (`nodes-frontend/`)
1. Перейдите в каталог и установите пакеты:
   ```bash
   cd nodes-frontend
   npm install
   ```
2. Создайте файл `.env.local` и укажите ключи вашего проекта Supabase:
   ```env
   VITE_SUPABASE_URL=https://your-supabase-url.supabase.co
   VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
   VITE_DJANGO_API_URL=http://127.0.0.1:8000/api/v1
   ```
3. Запустите dev-сервер:
   ```bash
   npm run dev
   ```

Для запуска обоих серверов одновременно можно использовать скрипт `run.bat` в корне репозитория.

## 📱 Мобильная разработка (Capacitor)

Для синхронизации веб-сборки с нативными проектами и запуска:

```bash

# Открыть Android Studio
npm run mobile:open:android

npm run mobile:open:ios
```
