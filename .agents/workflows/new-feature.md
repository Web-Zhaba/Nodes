# 🚀 Workflow: Новая Фича (Feature Cycle)

Используй этот воркфлоу для реализации любых крупных обновлений (например, "Public Sharing" или "Crystal Nodes").

## 1. Инициация (Brainstorming)
- [ ] Запусти навык `brainstorming`.
- [ ] Обсуди требования, дизайн и влияние на существующий граф.
- [ ] Создай тикет или запись в `docs/ROADMAP.md`.

## 2. Планирование (Writing Plan)
- [ ] Запусти навык `writing-plans`.
- [ ] Разбей задачу на атомарные шаги (Frontend, Backend, DB).
- [ ] Определи точки верификации.

## 3. Реализация (Execution)
- [ ] Следуй принципам FSD (Feature-Sliced Design).
- [ ] Используй `RULE[frontend.md]` для React:
  - Никаких `;`.
  - Только Tailwind.
  - Early returns.
- [ ] Используй `RULE[devops-backend.md]` для Django/Bash.

## 4. Тестирование (TDD)
- [ ] Если фича критическая — запусти `test-driven-development`.
- [ ] Напиши тесты для логики расчёта стабильности.

## 5. Завершение (Verification)
- [ ] Запусти навык `verification-before-completion`.
- [ ] Убедись, что все чекбоксы в плане отмечены.
- [ ] Сделай коммит по стандарту: `feat(scope): description`.

---
> [!NOTE]
> Каждая фича должна делать граф более "живым" и наглядным. Если она просто добавляет таблицу — подумай, как визуализировать это в Nodes.
