# 🕸 Workflow: Создание/Обновление узла (Node Creation)

В проекте Nodes "Узел" — это центральная сущность. Этот воркфлоу помогает пройти путь от идеи до реализации в коде.

## 1. Проектирование (Design)
- [ ] Определи тип узла: `BINARY`, `QUANTITY` или `DURATION`.
- [ ] Какая **Масса** у этого действия? (Как сильно оно влияет на стабильность ядра).
- [ ] Есть ли специфические метаданные? (Цели, интервалы).

## 2. Слой Данных (Data Layer)
- [ ] **Supabase**: Если нужны новые поля, обнови таблицу `nodes`.
- [ ] **Types**: Обнови `src/entities/node/types.ts`.
- [ ] **Store**: Убедись, что `useNodesStore` умеет обрабатывать новые данные.

## 3. Слой Логики (Logic Layer)
- [ ] **Django**: Если узел требует особой логики "остывания" или расчета стабильности, обнови аналитический сервис.
- [ ] **Zustand**: Реализуй экшены для создания импульсов (`Pulse`) для этого типа узла.

## 4. Слой UI (Presentation Layer)
- [ ] **Component**: Создай или обнови карточку узла в `src/widgets/node-card`.
- [ ] **Form**: Обнови форму создания узла (Zod schema + React Hook Form).
- [ ] **Graph**: Убедись, что `react-force-graph-2d` правильно визуализирует новый тип (цвет, размер).

## 5. Проверка (Validation)
- [ ] Создай тестовый узел.
- [ ] Отправь импульс.
- [ ] Убедись, что стабильность пересчиталась корректно.
